Claude Opus 4.8 Invents Schema Fields in Tool Calls

Developer Armin found that Claude Opus 4.8 — not a small model — was hallucinating extra, undefined keys inside the <code>edits[]</code> array of a structured tool call, causing Pi to reject the call and retry, degrading reliability despite the model getting the underlying edit right

So what

PMs building agentic pipelines should not assume that upgrading to a more capable model improves tool-use reliability; schema validation and graceful retry logic remain essential guard-rails, and this warrants explicit regression testing when swapping model versions
